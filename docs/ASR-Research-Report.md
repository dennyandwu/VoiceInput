## 一、竞品分析

### 1. SuperWhisper（$8-16/月）
- **架构**: macOS 原生 Swift
- **ASR**: Whisper（本地） + GPT-4o/Claude（云端后处理）
- **核心特点**:
  - 多模式系统：可按应用切换 mode（聊天/文档/法律）
  - 每个 mode 可独立选择 Voice Model（Whisper Large/Small）+ Language Model（GPT-5/Claude）
  - **"Super Mode"**: AI 感知屏幕内容，根据上下文优化输出
  - 自定义词汇表（Vocabulary）
  - Push-to-Talk
  - 文件转录
- **后处理**: 云端 LLM（GPT/Claude），口语→书面语转换
- **优势**: 最成熟的产品，模式系统灵活
- **劣势**: 付费，英文为主，中文不是重点

### 2. Typeless（医疗专用，已 pivot）
- **架构**: Electron + koffi（FFI）
- **ASR**: 本地模型通过 FFI 调用
- **后处理**: SQLite 词库 + 云端 LLM
- **核心特点**:
  - Squirrel 自动更新框架
  - SQLite + Drizzle ORM 本地数据库
  - 医疗术语专用词库
- **启示**: 词库系统 + LLM 后处理的组合方案

### 3. Open-Typeless（开源，参考项目）
- **架构**: Electron + Trellis 框架
- **ASR**: 火山引擎大模型 API（流式识别）
  - `volc.bigasr.sauc` — 大模型 1.0
  - `volc.seedasr.sauc` — 大模型 2.0（推荐）
- **文字注入**: `node-insert-text` + CGEvent
- **优势**: 开源可参考，流式识别体验好
- **劣势**: 依赖云端 API，无离线能力

## 三、中英混合识别方案

### 问题本质
中英混合（Code-Switching）是 ASR 领域的难题：
- 单语言模型对非目标语言准确率骤降
- 语言边界检测困难
- 不同语言的声学特征差异大

### 业界方案对比

| 方案 | 做法 | 延迟 | 效果 | 适合我们 |
|------|------|------|------|---------|
| **A. 单模型多语** | Whisper/SenseVoice 直接混识 | 最低 | 中 | ✅ 当前方案 |
| **B. 双引擎路由** | 语言检测→分别识别 | 中 | 中高 | ✅ v2.0 已实现 |
| **C. 云端 API** | gpt-4o-transcribe / SeedASR 2.0 | 高 | 最高 | ✅ 推荐增加 |
| **D. LLM 后处理** | 本地识别 + LLM 纠错 | 中高 | 高 | ✅ 推荐增加 |
| **E. 流式混合** | 实时流+云端，热词加权 | 最低 | 高 | ❌ 架构改动大 |

### 推荐组合策略

```
用户说话
  ↓
本地 SenseVoice（~50ms）→ 初步识别 + 语言检测
  ↓
├── 纯中文 → SenseVoice 结果 → 词库纠正 → 输出
├── 纯英文 → Whisper 结果 → 词库纠正 → 输出
└── 中英混合 → 云端 API（gpt-4o-transcribe）→ 输出
         或 → LLM 后处理（GPT-4o-mini 纠正）→ 输出
```

## 四、LLM 后处理方案

### SuperWhisper 的做法
SuperWhisper 的核心竞争力不是 ASR 本身（用的是标准 Whisper），而是 **LLM 后处理**：

```
Whisper 原始输出: "呃我觉得那个meeting呃大概是明天吧就是那个project review"
                 ↓ GPT-4o-mini 后处理
清理后输出: "我觉得 meeting 大概是明天，就是 project review。"
```

Prompt 示例:
```
你是语音转录的后处理助手。请将以下语音识别原始文本整理为干净的书面文本：
1. 去除口语填充词（呃、嗯、那个、就是、然后、like、you know）
2. 修正明显的语音识别错误
3. 添加正确的标点符号
4. 英文单词使用正确拼写和大小写
5. 保持原意不变，不要增加或删除内容
6. 中英文之间加空格

原始文本：{asr_output}
```

### 成本估算
- GPT-4o-mini: ~$0.15/1M tokens
- 平均一句话 50 tokens → 每次后处理成本 ≈ $0.00001（忽略不计）
- 每天说 200 句 → 月成本 < $0.10

### API 选择

| API | 延迟 | 价格 | 中文 | 推荐 |
|-----|------|------|------|------|
| **GPT-4o-mini** | 200-400ms | $0.15/1M | 好 | ⭐ 性价比最高 |
| **GPT-4o** | 300-600ms | $2.5/1M | 很好 | 高精度场景 |
| **Claude Haiku** | 200-400ms | $0.25/1M | 好 | 备选 |
| **Groq Llama** | <100ms | $0.05/1M | 中 | 最快 |
| **本地 Llama 3** | 100-300ms | 免费 | 中 | 离线备选 |

## 五、对 VoiceInput 的建议

### 短期（v2.x）
1. **增加 LLM 后处理选项**（方案 C + D 结合）
   - 菜单开关："AI 优化"（默认关闭）
   - 开启后：ASR 结果 → GPT-4o-mini → 清理输出
   - 用户填入 OpenAI API Key
   - 预估额外延迟 +200-400ms

2. **增加云端 ASR 选项**（针对中英混合）
   - 火山引擎 SeedASR 2.0（中文最佳）
   - OpenAI gpt-4o-transcribe（中英混合最佳）
   - 作为本地识别的补充，可选

3. **增强本地规则**
   - 口语填充词过滤（呃、嗯、那个）
   - 常见 ASR 错误自动修正词库预置

### 中期（v3.x）
4. **考虑 Fun-ASR-Nano**
   - 阿里 2025/12 新发布，千万小时训练数据
   - 31 语种，低延迟实时转录
   - 如果 sherpa-onnx 支持则可集成

5. **流式识别**
   - 当前是录完才识别，改为边说边出字
   - 体验提升巨大（参考 open-typeless 的火山引擎流式方案）

## 六、结论

**最优方案（推荐实施）：**

```
本地快速识别（SenseVoice + Whisper）
     +
可选 LLM 后处理（GPT-4o-mini，去填充词/纠错/格式化）
     +
可选 云端 ASR 兜底（gpt-4o-transcribe，中英混合最佳场景）
```

这就是 SuperWhisper 的核心架构：**本地 ASR + 云端 LLM 后处理**。
我们完全可以实现，而且成本几乎为零。
